## 本次会话：Thinking/Reasoning 流式显示修复

### 修改文件（未 commit）

**1. `frontend/src/core/threads/hooks.ts`**
- 第 258 行: `content: ""` → `content: t.common.thinking`
- 第 418 行: 依赖数组增加 `t.common.thinking`
- **原因**: glm-5 添加的乐观 thinking 占位消息 content 为空字符串，`groupMessages()` 中 `hasContent("")` 返回 false，消息被跳过永不渲染。用户发送问题后看不到任何"思考中"反馈。

**2. `frontend/src/core/messages/utils.ts`**
- 第 239-247 行: `hasContent()` 的 array 分支从 `array.length > 0` 改为 `.some()` 检查是否有真正的 text/image 元素
- **原因**: 旧逻辑对 `[{type: "thinking"}]` 也返回 true，导致 thinking-only 消息同时归入 `assistant:processing` 和 `assistant` 两个分组，thinking 内容重复渲染。

### 消息渲染流水线（关键架构）

```
hooks.ts 创建乐观占位（element:"task", _thinking:true）
  → useStream 连接 LangGraph 后端（streamSubgraphs:true, thinking_enabled, reasoning_effort）
  → 真实消息到达时清除乐观消息（messages.length 增加）

groupMessages (utils.ts) 分组逻辑:
  ai + hasReasoning/hasToolCalls → "assistant:processing" → MessageGroup（CoT UI）
  ai + hasContent + !hasToolCalls → "assistant" → MessageListItem
  两者可共存: 有 reasoning + text 的消息同时出现在两个分组

Thinking 提取优先级 (extractReasoningContentFromMessage):
  1. additional_kwargs.reasoning_content
  2. content[0].thinking (Anthropic 格式)
  3. <think>...</think> 内联标签
```
