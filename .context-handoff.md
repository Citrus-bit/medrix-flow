# MedrixFlow 项目上下文

## 项目信息
- **路径**: `REDACTED_PATH/Desktop/medrix-flow`
- **来源**: 基于 ByteDance DeerFlow 二次开发，已完成品牌定制
- **GitHub**: `https://github.com/Citrus-bit/medrix-flow`
- **本地访问**: `http://localhost:1000`

## 技术栈
- **后端**: Python 3.12, FastAPI, LangGraph, LangChain, uv
- **前端**: Next.js 16, React 19, TypeScript, TailwindCSS 4, Shadcn UI, pnpm
- **状态管理**: TanStack React Query（服务端状态）+ localStorage（本地设置）
- **部署**: Docker + nginx 反向代理

## 架构（4 个服务）
| 服务 | 端口 | 说明 |
|------|------|------|
| LangGraph | 2024 | Agent 编排引擎 (lead_agent) |
| Gateway API | 8001 | FastAPI 网关 (`backend/app/gateway/`) |
| Frontend | 3000 | Next.js (Turbopack) |
| Nginx | **1000** | 统一反向代理入口 |

## 启动
```bash
双击 start.command          # 一键启动所有服务 + 自动打开浏览器
make dev                    # 或手动启动
make dev-daemon             # 后台启动
make stop                   # 停止所有服务
```

### 模型卡片表单字段
- **模型** — 单一输入框，用户填写模型名（如 `gpt-4o`），同时作为 `name` 和 `model`
- **提供商** — Select 下拉，每个选项有唯一 `key`（openai / anthropic / google / deepseek / openai-compatible）
  - 选中状态用 ModelCard 内部 `presetKey` state 追踪，不从 provider 反推
  - 底层 `provider` 字段存储提供商类路径（如 `langchain_openai:ChatOpenAI`）
- **Base URL** / **API Key** / **Max Tokens** / **Temperature**
- **Thinking** / **Vision** 开关
- **测试按钮** — 动态实例化 provider 类发送 `ainvoke("Hi")`
- **API Key 眼睛按钮** — 输入框右侧外部 flex 布局，切换 password/text 类型

### 工具 Key 卡片
- Tavily / Jina 各一个密码输入框 + 眼睛按钮 + 测试按钮

### 保存流程
模型定义写入 `config.yaml`，API Key 写入 `.env`（config.yaml 中引用为 `$VAR`），自动 `reload_app_config()` 热重载

### Provider 预设（PROVIDER_PRESETS）
```typescript
[
  { key: "openai",            label: "OpenAI",            provider: "langchain_openai:ChatOpenAI" },
  { key: "anthropic",         label: "Anthropic",         provider: "langchain_anthropic:ChatAnthropic" },
  { key: "google",            label: "Google Gemini",     provider: "langchain_google_genai:ChatGoogleGenerativeAI" },
  { key: "deepseek",          label: "DeepSeek",          provider: "medrix_flow.models.patched_deepseek:PatchedChatDeepSeek" },
  { key: "openai-compatible", label: "OpenAI Compatible", provider: "langchain_openai:ChatOpenAI" },
```

注意：OpenAI 和 OpenAI Compatible 共享相同 provider 类路径，靠 `presetKey` state 区分。

## 注意事项
- demo 数据（`frontend/public/demo/threads/`）中保留原 DeerFlow 线程数据
- README 等文档中仍有 `your-org/medrix-flow` 占位符未替换为实际地址
- `medrix_flow.tech` 域名在部分脚本/文档中被引用，非真实域名
- 前端构建需要 `BETTER_AUTH_SECRET` 环境变量（`pnpm build` 时设置）
- Setup API 返回完整 API Key（无 mask），依赖本地网络安全（localhost 访问）
- **LangGraph dev 服务器线程持久化限制**: 使用 `langgraph_runtime_inmem`，线程元数据存内存 + pickle（`.langgraph_api/.langgraph_ops.pckl`），10 秒 flush 一次。服务重启或 pickle 损坏会丢失所有线程。SQLite checkpointer（`checkpoints.db`）只存 checkpoint 数据不存线程元数据。生产环境应使用 PostgreSQL 后端。

## 流式传输 & 线程持久化架构

### 流式传输管线
| 组件 | 说明 |
|------|------|
| `useStream` (SDK) | `@langchain/langgraph-sdk` React hook，管理 SSE 连接 |
| `reconnectOnMount: true` | 用 `sessionStorage` 存 `lg:stream:{threadId}` → `runId` 映射 |
| `streamResumable: true` | 断连后后端继续运行（`onDisconnect: "continue"`） |
| 乐观消息 | `thread.messages.length > prevMsgCountRef.current` 时清除 |

### 线程列表查询
- `useThreads()` → `threads.search()` POST `/threads/search`
- 默认参数: `limit: 50, sortBy: "updated_at", select: ["thread_id", "updated_at", "values", "status"]`
- `refetchOnWindowFocus: true`, `staleTime: 30_000`
- 新线程在 `onCreated` 回调中 optimistic 插入 query cache
- `visibilitychange` 监听器在 tab 恢复可见时 invalidate 缓存

### 思考/推理展示
- 发送消息后立即显示 `<Reasoning isStreaming={true}>` 组件（带脑图标 + "Thinking..." shimmer + 实时计秒）
- 由 `message.additional_kwargs._thinking` 标识触发（`message-list-item.tsx`）
- 真实推理内容到达后，由 `hasReasoning()` + `extractReasoningContentFromMessage()` 接管渲染

### 子代理任务展示
- `message-list.tsx`: 检测 `assistant:subagent` 消息组，从 `task` tool_calls 创建 `Subtask` 对象
- `subtask-card.tsx`: ChainOfThought 卡片，shimmer 描述文字、ShineBorder 动画、实时 tool_call 进度
- `task_running` 自定义事件通过 `onCustomEvent` → `useUpdateSubtask()` 更新 SubtaskCard
- `core/tasks/context.tsx`: SubtaskContext 管理任务状态，`setTasks` 仅在 `latestMessage` 存在时触发重渲染
